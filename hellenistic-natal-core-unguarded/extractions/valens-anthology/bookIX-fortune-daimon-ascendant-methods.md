# Valens Extraction - Book IX Fortune, Daimon, And Ascendant Methods

Source: Vettius Valens, `Anthologies`, Book IX, Mark Riley translation.

Range: local PDF pp. 151-166.

Brennan cross-ref: `[cross-ref: Brennan Ch.10]`, `[cross-ref: Brennan Ch.16]`, `[cross-ref: Brennan Ch.17]`, `[cross-ref: Brennan Ch.18]`.

Copyright note: this file paraphrases method and stores page references. It does not reproduce long passages.

## Method Takeaway

Book IX revisits Fortune, Daimon, the twelve places, Moon-based timing, illness initiatives, Ascendant-degree calculation, nodes, conception, and data precision. It also contains methodological reflection on authorities, tables, fate, and the limits of human control. The strongest project-level uses are Fortune/Daimon timing support, place activation, data accuracy discipline, and ethical containment for specialist material.

## Rule Card IX.1 - Fortune And Daimon Are Powerful Timing Anchors

Source: Book IX.1K;2P, local PDF pp. 151-153.

Use when:

- timing body/circumstance through Fortune;
- timing action/undertaking through Daimon;
- comparing Lot-derived chronocratorships.

Required chart data:

- Fortune and Daimon positions;
- rulers and conditions;
- time-lord method;
- configured planets.

Procedure:

1. Calculate or import Fortune and Daimon.
2. Choose the Lot that matches the topic.
3. Audit the Lot ruler and configured planets.
4. Use the Lot as timing anchor only inside an explicit method.

Guardrail:

Fortune/Daimon timing must remain distinct from ordinary Lot mention in natal synthesis.

Destination rule file:

- `rules/lots-and-fortune.md`
- `rules/timing-techniques.md`

Provenance: `[Valens *Anth.* IX.1K;2P, PDF pp. 151-153]` `[cross-ref: Brennan Ch.16]`.

## Rule Card IX.2 - Places Can Be Activated In Propitious/Impropitious Times

Source: Book IX.2K;3P, local PDF pp. 153-154.

Use when:

- timing a specific house/place topic;
- the time-lord method activates a place rather than only a planet;
- comparing place topic with natal promise.

Required chart data:

- active place;
- ruler of active place;
- planets in or configured to it;
- timing anchor.

Procedure:

1. Identify the active place in the timing method.
2. Recall its natal topic field.
3. Audit the ruler and planets connected to that place.
4. Interpret the period as activating that place's natal condition.

Guardrail:

Place activation is not the same as a permanent natal emphasis. It is time-bound.

## Rule Card IX.4 - Critical Times Depend On Contacts And Unions

Source: Book IX.4K;5P, local PDF p. 155.

## Rule Card IX.6 - Ascendant Degree Methods Are Data-Quality Tools

Source: Book IX.6K;7P, IX.10K;11P, IX.18-IX.19, local PDF pp. 156-161, 165-166.

- checking sensitivity to birth hour;
- documenting why exact time matters.

Required chart data:

- birth time and fraction of hour;
- Sun and Moon positions;
- table/factor method;

Procedure:

1. Identify the Ascendant calculation method.
2. Record all input assumptions.
3. Compare the derived Ascendant with supplied chart data.

## Rule Card IX.9 - Exact Hour/Fraction Is A Hard Data Gate

Source: Book IX.15, local PDF p. 163.

Use when:

- a method requires exact Ascendant or Moon degree;
- the birth time is rounded or uncertain;
- deciding whether to run a timing/rectification method.

Required chart data:

- birth time precision;
- Ascendant degree sensitivity;
- Moon motion if required.

Procedure:

1. Record time precision.
2. Identify methods blocked by insufficient precision.
3. Mark `data-limited` rather than guessing.
4. Offer only methods that can tolerate the available data.

Guardrail:

No exact-time method should run on fuzzy time data without an uncertainty note.
