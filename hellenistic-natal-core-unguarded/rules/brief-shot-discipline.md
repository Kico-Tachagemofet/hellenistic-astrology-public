## 4.1.3.1 Definition

A **brief shot** is a piece of natural-language output that grounds an abstract
astrological structure into concrete real-world domains the reader can
recognize. It is a **multi-alternative concrete category** statement, not a
single-moment sensory scene.

Each brief shot must surface ≥2–3 specific category alternatives, each
attached to an industry / topic / relationship type / domain example, each
backed by a chart-factor evidence cite, plus an explicit not-fit segment,
maintained hedging, and subject discipline. (Full criteria list in §4.1.3.2.)

Brief shots belong to the **output surface** of a reading. The
evidence-chain audit (per `reading-workflow.md`) produces the structural
finding; the brief shot translates that finding into the specific category
landscape it implies, so the reader can self-identify rather than be told
abstract typology that fits anyone.

> **Distinction**: brief shot ≠ sensory-anchored vignette. See §4.1.3.6.

## 4.1.3.2 Required Criteria

Every brief shot must satisfy **all 8** criteria. Missing any one means the
brief shot is non-compliant and must be revised before output.

1. **≥2–3 specific category alternatives**
   - At least two, preferably three, parallel categories (labelled A / B /
     C... or comparable).
   - Single-option output ("适合 X") fails this criterion.

2. **Each alternative attached to industry / topic / type example**
   - 不止类名 — each category names concrete industries, professions,
     relationship types, study fields, income paths, family dynamics,
     occult traditions, or domains. The reader should be able to point at
     a recognizable real-world instance.

3. **Evidence-cite per alternative**
   - Each alternative is followed by "为什么 fit"（chart factor + rule
     cite）— e.g., "Venus 9 exalted + Mercury Sag wide → 适合 cross-cultural
     advisory". Cites can be parenthetical or in a "为什么" tail; they
     anchor the category to the underlying structural finding.

4. **Not-fit segment (≥1 explicit "不太适合 X")**
   - At least one explicit "不适合 / 不太适合 X" entry per finding, with its
     own evidence cite. Helps the reader self-identify by ruling things out,
     not only by inclusion.

5. **Conditional 进阶 (optional but recommended)**
   - Where applicable, add "如果配合 Y 因素，可能 Z" entries that let the
     reader evaluate sub-conditions themselves. Not strictly required for
     every brief shot, but recommended whenever the structural finding has
     a known sub-condition that flips or escalates the reading.

6. **Hedging maintained**
   - "倾向 / 可能 / likely / 偏向" — never "会 X" / "一定 X" / 断言. The
     brief shot is a family of plausible forms, not a prediction.

7. **Subject = person / state / behavior**
   - The grammatical subject of every life claim in the brief shot is the
     native, a person, an event, a state, or a life domain — never a chart
     factor. "案主在跨文化场域 likely 找到 footing" — yes. "Venus 9 宫
     means cross-cultural fit" — no (chart-factor subject).

8. **No astrological term in brief shot text**
   - "星座 / 宫位 / 度数 / Mars / Venus / ruler" 等 technical terms 不
     appear in the brief shot prose itself. Evidence cite **is** acceptable
     in parenthesis or footnote (it is a citation, not a claim subject).
   - Example: "适合 cross-cultural advisory（Venus 9 exalted + Mercury
     Sag wide）" — the parenthetical is a cite, the main clause is plain.

## 4.1.3.3 Anti-Patterns (Avoid)

Each anti-pattern below corresponds to a criterion failure in §4.1.3.2. If a
draft brief shot matches any pattern, revise.

- **Generic typology** — "适合做与人打交道的工作" / "可能在感情中遇到挑战".
  Can be applied to anyone; carries no information. (Violates #1, #2.)
- **Single option** — "适合 X 工作" with no alternatives. (Violates #1.)
- **No evidence cite** — "是这样的" / "适合 Y" 不解释 why. (Violates #3.)
- **No not-fit segment** — only inclusive examples, no exclusions; reader
  cannot self-rule-out. (Violates #4.)
- **Over-assertion** — "会 X 发生" / "一定 X" 而不是 "可能 X" / "倾向 X".
  (Violates #6.)
- **Astrological term leakage in output text** — "7 宫主 暗示..." / "Mars
  在 10 宫意味着..." in the brief shot prose itself. Cite belongs in
  parenthesis / footnote, not as the subject or main predicate. (Violates
  #8.)
- **Chart factor as subject** — "Sun 7 宫 means the native..." / "Mars own
  domicile gives 案主 work drive". Chart factor must not be the
  grammatical subject of a life claim. (Violates #7.)
- **Doom framing** — "命主会陷入 X 不幸" — independently violates the
  difficult-condition guardrail in `delineation-output-rules.md`; the brief
  shot must state pressure + mitigation + concrete channel, not a verdict.

## 4.1.3.6 Language Convention (technical layer vs TL layer)

Brief shots surface at two registers — **technical** (Stage 2 topic modules
+ `delineation-output-rules.md` Synthesis technical anchor) and **TL**
(translation-layer pass output). The 8 criteria in §4.1.3.2 apply
identically at both registers; only the surface language differs.

- **Technical layer brief shot — English category names.**
  Per v2.2 architecture rule, Stage 1 (data extraction) + Stage 2 (topic
  modules) + Stage 3 (synthesis) analysis output is written in **English**.
  LLM internal evidence-chaining is more precise in English, and
  Hellenistic technical terms (reception / dignity / aspect / sect) keep a
  stable single-form referent without 中英切换 friction. Brief shots
  embedded in technical Stage 2 module output use English category names +
  English evidence cites.

- **TL layer brief shot — full Chinese category names.**
  TL is the reader-facing deliverable. Brief shot category names (具体行业
  / 关系类型 / 场景类别) are rendered **fully in Chinese** in the main
  clause. Evidence-cite parenthesis MAY retain English technical terms
  (e.g., "Venus 9 exalted", "Mercury R 6") for traceability back to Stage
  1 / 2 tables — that is a citation, not a category label.

**Side-by-side example (same finding, two registers):**

| Register | Brief shot |
|---|---|
| **Technical layer** | "Consulting (management / policy / education / occult-divinatory)" |
| **TL layer** | "咨询业（管理咨询 / 政策智库 / 教育咨询 / occult/divinatory 咨询）" |

Both satisfy §4.1.3.2 in full; only the surface language register changes.

- **Existing cases preserved as-is.** `cases/case-01-*` through

- **Convention is broader than brief shots.** The English-for-technical /

  governs non-brief-shot output at the same surfaces (Stage 1 / 2 / 3
  analysis = English; TL pass = Chinese). This file documents the

## 4.1.3.7 Discipline vs Sensory Vignette Distinction

Brief shot is **not** a sensory-anchored vignette. They are different
grounding registers, and only the brief shot is what this discipline
governs.

- **Brief shot**: "适合 A 顾问 / 咨询 / 智库类（管理咨询、政策智库、法律
  咨询...）；B 教育 / 学术 / 出版类（高校教师、研究员、学术编辑...）；
  不太适合 高节奏销售 / 短决策金融." — multi-alternative concrete
  category landscape.
- **Sensory vignette**: "Wednesday morning when the alarm rings, the
  decision to go to the office or stay home feels heavier than usual..."
  — single-moment scene with sensory texture.

Vignette frame is a legitimate grounding mode for some output contexts and
is **neither required nor forbidden** by this file. It just isn't what
"brief shot" means here.

The brief shot's main deliverable is the question:
**"这种结构在现实生活里 likely 对应什么 specific industry / relationship
type / subject domain?"** — not the question "这种结构感受起来像什么
sensory moment?".

If a session produces a vignette where a brief shot is required (Stage 2
topic module, TL Multi-Example, etc.), it has not satisfied the discipline,
even if the vignette is well-written.

## 4.1.3.8 Provenance

- **At discipline level**: `[project heuristic]`. The 8-criteria structure,
  the not-fit gate, and the explicit "no astrological term in output text"
  rule are project-specific output discipline, not distilled from a single
  source.
- **Reasoning grounded in traditional Hellenistic delineation form**:
  classical practitioners (Lilly's *Christian Astrology* judgments, Bonatti's
  considerations, Valens's case examples) routinely list 多例式 — several
  alternative occupations / relationship types / outcomes — with explicit
  reasoning trails, rather than collapsing to single-typology generalities.
  Brief shot discipline restates that classical practice in modern
  output-rule form.
- **8 criteria 与 范文 §V 一致**: the criteria in §4.1.3.2 are the formal
  statement of §V.5's "合规 brief shot 要素" / "不合规 brief shot 特征"
  pattern summary. Any future revision to §4.1.3.2 must reflect back into
  范文 §V, and vice versa.

## 4.1.3.10 范文 Maintenance / Iteration Policy (C.5 deliverable)

This section defines how `samples/topic-reading-template.md`
(the v2.2 technical-layer canonical) is maintained across audit /
iteration cycles. Goal: keep the 范文 a stable reference for
sub-session dispatch while allowing demonstrable improvement.

### Trigger conditions for 范文 update

A 范文 update is justified only by:

1. **Audit session finding** — a dispatched audit session finds
   demonstrable error (deterministic data error, structural finding
   inconsistency, register-convention violation, brief shot
   8-criteria violation, etc.). Pass 1 (17 errors + 1 cascade)
   and Pass 2 (register refactor) are the canonical precedents.
2. **Plan A / B / C spec drift** — execution of any v2.2 phase
   surfaces a spec ↔ 范文 inconsistency (e.g., B.5 spec'd a
   sub-section structure 范文 §IV did not match → Pass 2 split
   resolved).
3. **New topic module added** — per `rules/topic-modules.md` §4.1.3
   extension, when a new module's brief shots and lens reframing
   need canonical illustration.
4. **Brief shot discipline refinement** — when this file
   (`rules/brief-shot-discipline.md`) is materially extended,
   范文 §III / §V examples may need to mirror the refinement.

Casual edits (typos, formatting) **do not** require Plan
scaffolding — they go through a single commit with explanation in
the commit message.

### Update procedure

- **Structural changes** (new sub-section / re-register / split /
  merge across sections) → start a `v2.2.x` or `v2.3.x` patch plan
  document; refactor sub-session dispatch via handoff with audit
  gate explicit on byte-equivalence of preserved content.
- **Minor changes** (typo, clarification, single-cite update) →
  single inline commit; commit message describes change scope.

### Versioning convention

- 范文 顶部 audit pass history blockquote 列每 audit pass entry
  (date + scope + cite to relevant Plan or audit session). Pass 1
  + Pass 2 已 set the format.
- Significant rewrites (>30% line churn or structural re-register):
  before overwriting, archive previous as `samples/topic-reading-
  template-vN.md` (N = sequential integer; N=1 for the pre-Pass-1
  version, etc.) — never overwrite the current canonical without
  archive when the change crosses the structural-vs-minor boundary.
  Prospective only: Pass 1 + Pass 2 were additive-revising the
  same file (no archive cut); the archive pattern applies to
  future structural rewrites that cross this boundary.
- All 范文 updates respect AGENTS.md §"v1 修改协议" additive-first
  discipline + watermark format.

### Audit signals routing

When a dispatched audit session finds an issue, route the fix per
issue class:

- **Geometric / dignity / reception errors** → 范文 §II ground-truth
  fix + 反查 `rules/stage1-data-extraction.md` Procedure (Plan A
  spec) to see if procedure tightening is needed (per Pass 1's
  17-error → A.2 Gate 4 lookup-verified pass linkage).
- **Brief shot specificity issues** → 范文 §III / §V example fix +
  `rules/brief-shot-discipline.md` 8 criteria refinement if pattern
  generalizes (per C.1 §4.1.3.2 + §4.1.3.3).
- **Lens reframing issues** → 范文 §III lens table fix + `rules/
  topic-modules.md` §"Topic: <slug>" spec re-check (per B.2).
- **Module structure issues** → 范文 §III sub-section fix + `rules/
  topic-modules.md` framework (§4.1.3 / §"Stage 2 Session Output
  Format" / §"Stage 3 Synthesis Spec") re-check (per B.1 / B.4 / B.5).
- **Register hygiene issues** (CN-EN mixed where spec mandates
  single register) → 范文 register cleanup + this section + `rules/
  aspect-reception-rules.md` §"Reception Detection Procedure" §
  "Language convention" (A.1) + `rules/topic-modules.md` §"Framework"
  preamble cross-cite (per Pass 2 precedent).
