# Fictional / Canon-Marked Subject Ethical Guard — REMOVED in this fork

> v2.3-unguarded (2026-05-18) — this rule file is a **stub** in the
> hellenistic-natal-core-unguarded fork. The original ethical-guard
> framework (5-reason exclusion table + body/illness/death/longevity
> default-OFF for real-person subjects + per-subject-class attenuation
> table + canon-resonance speculative downgrade) has been removed per
> practitioner feedback on case-10. See guarded variant
> `../../hellenistic-natal-core/rules/fictional-subject-ethical-guard.md`
> for the original framework.

## What is removed

| Original mechanism | Removed effect |
|---|---|
| Subject classification (real person / public figure / fictional+canon / hypothetical) | All subjects processed identically; no per-class branching |
| 5 exclusion reasons (triangulation impossible / single-source / tradition weak / birth-time precision / disease-death-social-prediction / modern revival) | Reasons 1/2/3/6 remain as **confidence-tier evidence inputs** (their original use), NOT as topic exclusions |
| Default-OFF on body / illness / death / longevity topics for real-person subjects | All 15 topics default-ON (per SKILL.md Step 4) |
| Per-reason attenuation table for fictional+canon subjects | Not needed — no per-class branching |
| `confidence:speculative` auto-flag on body/death/longevity findings | Confidence tier follows actual evidence merit, not topic membership |
| Canon-resonance findings auto-downgraded to speculative | Canon-resonance findings get the tier their evidence supports |
| "symbolic resonance" / "character resonance" required framing | Removed — direct relational/event-level framing per `delineation-output-rules.md` Direct Relational Statement section |

## What is kept

- **≥2-source triangulation evidence standard** — kept (this is doctrinal
  evidence quality, not ethical gating; lives in `delineation-output-rules.md`
  Evidence Standard)
- **`[Source-only candidate]` flag** — kept (data quality marker)
- **Picatrix / Agrippa operative magical procedure SCOPE-out** — kept
  (this is project scope, not subject ethics; lives in SKILL.md
  Trigger conditions "Do NOT trigger" list)
- **Stage 1 birth-time precision requirements** — kept (data contract,
  not ethics)

## How findings now treat former-extension topics

Children / siblings / friends-benefactors / illness / death / longevity /
body — all 7 are now ordinary topics. Their findings:

- Use the normal Hellenistic significator chain (per
  `significator-hierarchy.md` + topic Lots per `lots-and-fortune.md`)
- Use the normal confidence tier scale (High / Moderate / Low /
  Speculative) per actual evidence — body/death/longevity do NOT
  auto-downgrade to Speculative
- Use direct relational/event-level statements per
  `delineation-output-rules.md` Direct Relational Statement section —
  no "symbolic framing only" hedge, no "mode language" abstraction
- Carry the same Stage 1 row_id evidence chain as any other topic
- Carry the same source-status-labeled tags from
  `chart-finding-index-tags.md`

## Force / Power signature framing

For chart-level archetype work — the lord-of-the-geniture, planetary
signature, most dignified / most ill-dignified of-sect, overcoming
patterns, cluster signatures, Mars/Saturn/Venus archetypal weight — all
remain methodologically grounded per `significator-hierarchy.md` +
`aspect-reception-rules.md` + `dignities-reference.md`. The
"attenuated reason 5" guard from the guarded variant is moot here
because there's no longer a guard to attenuate.
