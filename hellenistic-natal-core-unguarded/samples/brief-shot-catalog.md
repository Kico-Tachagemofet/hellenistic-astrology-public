## Purpose

Reference catalog of **compliant brief shot examples** (per topic, per
finding theme, in both technical and TL register) plus an **anti-pattern
catalog** of common non-compliant typology with discipline-violation
annotations. Use as a quality benchmark when writing Stage 2 topic
module output or TL pass output. Supplements the 范文 references in
`rules/brief-shot-discipline.md` §4.1.3.4 by extending coverage:
范文 §V provides 4 topic modules; this catalog extends to all 7
defaults with 3–5 finding themes each, plus cross-topic theme
applications and a dedicated anti-pattern section.

This file is examples; the spec — the 8 criteria
(`rules/brief-shot-discipline.md` §4.1.3.2), the 8 anti-patterns
(§4.1.3.3), the application contexts (§4.1.3.5), and the register
convention (§4.1.3.6) — lives in the discipline file. If a catalog
example and the spec ever disagree, the spec wins; this file should be
revised to match.

---

## How to use

When you are about to write a brief shot for a Stage 2 topic module
output or a TL pass output:

1. **Identify topic + finding.** Which topic module is this brief shot
   for (career / love / etc.)? What structural finding from Stage 1 /
   Stage 2 lens does it concretize?
2. **Check the catalog for similar finding themes.** Locate the topic
   section below; scan the 3–5 finding themes for one whose structural
   anchor parallels yours.
3. **Match the catalog format.** Your brief shot should mirror the
   matched theme's shape: ≥2–3 alternatives, evidence cite per
   alternative, ≥1 explicit not-fit segment, hedging, optional
   conditional escalation.
4. **Self-check against the 8 criteria in
   `rules/brief-shot-discipline.md` §4.1.3.2.** Every criterion must be
   met. Missing any one means revise before output.
5. **If register is unclear, sample 2–3 catalog entries on the same
   topic.** Compare technical-register and TL-register blocks
   side-by-side to confirm category-name register, evidence-cite
   convention, and not-fit framing match your output surface.

If you are auditing existing output (rather than writing fresh), the
anti-pattern section is the faster entry point: scan the typology
examples for shape matches to your draft, then trace back to the
specific §4.1.3.3 anti-pattern number cited.

## Register convention

Per `rules/brief-shot-discipline.md` §4.1.3.6, the 8 criteria are
register-neutral; only the **surface language** of category names
differs between the two registers:

- **Technical register** (Stage 2 topic module output) uses **English**
  category names in the main clause. Evidence cite parentheses may keep
  Hellenistic technical terms (Venus, 9, exalted, Mercury R, ctos,
  etc.) — those are citations, not category labels.
- **TL register** (TL pass output) uses **full Chinese** category names
  in the main clause. Evidence cite parentheses MAY retain English /
  Hellenistic terms for traceability back to Stage 1 / Stage 2 tables.

Each topic section below provides **both registers** per finding
theme. The anti-pattern section is register-neutral structurally —
bad examples may appear in either language; the anti-pattern itself
is the violation type, not the surface language.

Most examples in this catalog are grounded in the same sample chart
that `samples/topic-reading-template.md` works on (Asc Cancer, Moon own
1, Mercury R 6 Sag, Mars own 10 Ari, Venus exalted 9 Pis, Saturn 5
Sco peregrine ctos, Jupiter 4 Lib ctos cadent, Sun ctos 6 Sag). Where
a finding theme illustrates a structure that does **not** appear in
the sample chart, the seed annotation explicitly marks it as
`[catalog extension — hypothetical structure for breadth coverage]`.
