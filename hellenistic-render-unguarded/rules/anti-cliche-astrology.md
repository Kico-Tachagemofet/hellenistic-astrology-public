# 反八股 — 占星 reader 文体打磨清单

> 从 anti-cliche.md 通用创意写作规则适配为占星 reader 渲染场景.
> 用途: Step 3 叙事段写完后 self-review, 不是写作前读.
> 前提: Step 1.5 生活判断摘要已完成, 叙事已落地为可验证判断.
> 本文件做的是文体微调, 不是 register 转换.

---

## E. 禁刚性句式

### E.1 禁"不是…而是…"及一切机械对照

占星渲染里这个句式的典型出现方式:

```
❌ "这不是冷漠, 而是结构性疏远"
❌ "不是没有自我, 是自我之主住在对面"
❌ "不是意志力缺了, 是这条线结构上就差最后一截"
❌ "不是被边缘的, 是家里的轴"
❌ "不是温情包围那种粘, 是默认就在场的那种粘"
```

全部改成直接说后半句:

```
✅ "父亲是疏远的, 不参与日常"
✅ "独处的时候找不到自己, 对面有人才定得住"
✅ "逼自己做计划总坚持不过三天"
```

同样禁止的变体:
- "并非…而是…" / "与其说…不如说…"
- "既…又…" / "像…又像…"
- "不要…要…" / "没有…只是…"
- "可以…但不…"

### E.2 禁解释尾巴

占星渲染里最常见的解释尾巴:

```
❌ "土星落陷在 6 室, 这意味着承担会特别重"
❌ "月亮合土星, 这显示出情绪底色偏沉"
❌ "金星四层尊贵, 为整盘增添了一个稳定的锚点"
❌ "火星被烧, 这是某种自我消耗的信号"
```

直接写判断, 删掉连接词:

```
✅ "承担特别重"
✅ "情绪底色偏沉"
✅ "审美判断力天生靠谱"
✅ "找到自己的过程默认就费劲"
```

判断本身已经在说话, 不需要再解释判断在说什么.

### E.3 禁假对立 / 空情感修饰

占星渲染里的典型假对立:

```
❌ "近乎温柔的承担"
❌ "介于亲密与疏远之间"
❌ "带着某种沉默的力量"
❌ "一种说不清的厚重感"
```

要么说清楚, 要么不说:

```
✅ "母亲是粘的——默认就在场, 离不开, 但也总擦火"
✅ "父亲扛了一些重的东西, 但你跟他从来不直接谈"
```

### E.4 禁排比列举

占星渲染里排比的常见形式:

```
❌ "在家里容易点火, 在公司容易点火, 在朋友面前也容易点火"
❌ "他看见了父亲的沉默, 他看见了母亲的粘腻, 他看见了自己的愤怒"
❌ "有承担, 有摩擦, 也有兜底"
```

合并成一句或者只说最重要的那一条:

```
✅ "在家人面前点火快, 比对外人频率高得多"
✅ "家里冷 + 粘 + 远 + 重这一组同时在"
```

## F. 抗重复

### F.1 同一个 chart fact 不换说法重复

占星渲染最常见的重复: 一个配置用三种说法说同一件事.

```
❌ 第一段: "自我之主在对面"
   第二段: "找到自己要通过对面的人"
   第三段: "独处时自我感不容易聚焦"
   → 三句话说的是同一件事 (Mars 7H), 只需要留最具体的一句
```

判定标准: 如果把某句话删掉, 读者丢失了什么新信息吗? 没有就删.

### F.2 段落结构不复用

8 段 per-planet 结构特别容易出现模板化:

```
❌ 每段都是: "XX 在这个人身上表现为…" → 正面 → 反面 → "所以…"
```

每段找不同的切入方式. 有的段从场景切入, 有的从时间切入, 有的从关系切入, 有的从一个具体判断直接开始.

### F.3 不复制前文用过的短语

跨段检查: "默认就是""结构上""不是…是…""对面那个人"——如果某个短语在前面段落用过, 这段换一种说法或者直接不说.

## G. 句子纪律

### G.1 每句话必须做工

写完一段回头读, 逐句问: 这句话推进了什么新判断? 如果拿掉它前后没有缺失, 删掉.

占星渲染里最常见的不做工句子:

```
❌ "这是这张盘里很重要的一个配置" → 不做工, 删
❌ "具体来说" / "换句话说" / "拆开说" → 连接词, 大多数时候可以删
❌ "这不是说一定会怎样" → 不做工的 hedge, 删
```

### G.2 不堆砌

```
❌ "深深的、沉默的、结构性的、长期的承担" → 四个形容词堆在一起
✅ "承担很重, 会持续很久" → 够了
```

### G.3 因果链清晰

时间节点和判断之间的因果必须读得通:

```
❌ "2028 年事业会有变化, 因为走到了 10 宫" → 读者不知道"走到 10 宫"为什么导致变化
✅ "2028 年事业方向会重新定, 因为那年管事业的那颗星被激活了, 同时管钱和社交的线也一起动"
```

## H. 抗短句碎片化

占星渲染里短句碎片化的典型:

```
❌ "父亲是远的. 母亲是粘的. 父母对立. 家里有旧账."
   → 四个短句切成四块, 读起来像 PPT 提纲
```

```
✅ "父亲是远的, 母亲是粘的, 父母彼此对立但不分手, 家里有从长辈
   那代传下来的说不清的旧账沉在底下."
   → 一口气读完, 信息密度一样但节奏连贯
```

短句单独成行只在需要冲击力的时候用:

```
✅ 前面铺了一段之后:
   "这个人不会真的散架."
   → 单句收束, 有冲击力, 可以用
```

## I. 别"不是而是"了（最高优先级, 跟 E.1 同一条铁律）

写完全文搜一遍 "不是" "而是" "并非" "与其说", 找到就改. 直接说后半句.

占星渲染里这个句式出现频率极高, 因为模型习惯先树一个"读者可能的误解"再纠正. 但大多数时候那个误解是模型自己想象出来的, 读者根本不会那样读.

```
❌ "这不是说他一定家里很苦, 是这张盘里的家庭路径就是冷+粘+远+重这一组"
✅ "这张盘里的家庭路径就是冷+粘+远+重这一组"
   → 前半句"这不是说他一定家里很苦"是在打一个没人立过的靶子, 删掉
```

## J. 写完停一下

每段写完, 不急着写下一段. 回头把这段默读一遍, 问三件事:

1. 有没有哪句话其实是在重复前面某句话换了个说法?
2. 有没有哪句话读者需要"翻译"才能验证准不准?
3. 有没有"不是而是"溜进来?

有就改. 三件事都没有才往下走.
