# 术语渲染纪律 — 三层 register

> 管 reader 正文（中文散文）和技术依据（trailing block）两层的术语处理。
> 核心原则：**reader 正文的任务是翻成生活判断，不是翻成中文术语**。
> 术语表不是逐词替换指令，是三层 gate——技术层有固定译名、reader 层有轻术语白名单、其余必须改写成场景/体感/行为描述。

---

## 第二层：reader 正文允许出现的轻术语

这些词可以在 reader 中文叙事段出现，但**必须跟一句人话**。不能干扔术语当结论。

**白名单**（允许出现 + 必须配人话）：

- 命主（上升之主）
- 福点 / 灵点
- 小限年 / 年主星
- 互容
- 落陷 / 失势 / 擢升 / 入庙
- 焦伤 / 被太阳烧
- 角宫 / 果宫
- 合相 / 对冲 / 六合 / 三合 / 刑
- 星座释法 / 一级期 / 二级期
- 解缚
- 行星年

**用法示范**：

```
✅ 火星是你的命主——管你上升天蝎的那颗，落在 7 室被太阳烧着.
   你很难靠独处把自我感立起来，关系一出现才容易知道自己要往哪走.

❌ 命主落陷焦伤在果宫互不相见.（干术语堆叠，没人话）
```

## 第三层：reader 正文禁止逐词翻译，必须改写成生活判断

这些词在 reader 中文叙事段**不能出现为术语**（不管中文还是英文形式）。必须翻成场景、体感、行为模式、关系描述、事件类型。

### 非术语高频词（英文混入叙事是 case-09 v1 最大可读性问题）

| 禁止出现 | 必须改写成 |
|---|---|
| baseline / 底色 / 默认状态 | 描述具体什么在长期运行，例："身体那一线长期偏重" |
| backdrop | 同上 |
| backbone / spine / 骨干 | 描述具体什么跟什么连在一起，例："前台的火和后方的金是同一根轴的两端" |
| carrier / 承载者 | 说谁扛了什么，例："土星在扛你接下来 27 年身体和生计的重" |
| peak / 最浓段 | 说具体时间段 + 什么变浓，例："2024 年底到 2027 年 2 月身体那条线最压人" |
| routing / route through / 走…线 | 说具体经过什么到达什么，例："公开形象要借金星才发得出去" |
| channel / vehicle | 同上 |
| bridge / 桥 | 说两件事怎么连上的，例："火星跟金星互容，前台和后方才不断" |
| anchor / 锚 | 说具体什么撑住什么，例："木星本宫本旬撑住整盘的归途" |
| load / stack / cluster | 说具体几件事压在一起，例："三颗星挤在 6 室白羊，月亮水星土星全在" |
| lock / compression | 说具体什么被卡住，例："三颗星互相看不见上升，力量出不来" |
| window | 说具体时间段 + 什么打开，例："8 月底到明年 8 月方向感和创造力在一生里最被放大的 12 个月" |
| track / axis | 说具体哪两件事分开跑，例："方向感那边亮，身体那边重，两边不互相抵消" |
| foreground / background | 说具体什么在台前什么在后面，例："身体的重从后台推到前台" |
| split | 说具体什么跟什么分开，例："你的公开面和内在感受对不上，两边都成立" |
| bifurcation | 同 split |
| concurrent | 说同时，例："同一段时间里" |

### 术语也不能干扔

| 禁止形式 | 必须改写成 |
|---|---|
| "Saturn 27 年 backdrop" | "从 2024 年底到 2051 年，身体、日常生计和现实承担这一线长期变重" |
| "Asc-Lord combust in 7H" | "你很难靠独处把自我感立起来，关系和合作一出现才容易知道自己要往哪走" |
| "dignity stack / quadruple-stacked" | "金星在这张盘里处在最好的位置——四层尊贵叠在一起" |
| "dispositor terminus" | "整盘所有行星的归途最终都收在木星身上" |
| "MR bridge" | "火星跟金星互容——整盘唯一一对既互接又看得见对方的" |
| "Mars 12-fold" | "火星身上压了整盘最多的角色——12 件事挂在它一颗上" |
| "of-sect malefic" | "火星在这张夜盘里是自己这一派的，破坏力打折" |
| "aversion with reception" | "隔了一面墙看不见对方，但在 dignity 层互相接住了" |

## 应用优先级

1. **先想这句话要告诉 reader 什么生活判断**
2. **用第三层的改写方式落地**
3. **如果生活判断必须带一个轻术语做锚**（例："你的小限年走到 5 室"），用第二层白名单 + 跟一句人话
4. **trailing 技术依据用第一层固定译名**，跟 source 对齐

reader 段永远不应该读起来像"翻译过的技术文档"。应该读起来像一个懂这些的人，坐下来跟你说你这辈子大概是怎么回事。
